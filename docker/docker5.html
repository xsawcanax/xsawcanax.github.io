Korzystanie z rejestrów, repozytoriów i znaczników obrazów Docker

1. Rejestr (Docker Registry):

Definicja: Rejestr Dockerowy to centralne repozytorium do przechowywania i udostępniania obrazów Dockerowych.

Przykład: Docker Hub, Google Container Registry (GCR), Amazon Elastic Container Registry (ECR), własny prywatny rejestr.

2. Repozytorium (Docker Repository):

Definicja: Repozytorium Dockerowe to kolekcja powiązanych ze sobą obrazów Dockerowych, zwykle różniących się znacznikami.

Przykład: W Docker Hub, repozytoria to nazwy użytkowników lub organizacji, a w nich znajdują się różne obrazy.

3. Obraz (Docker Image):

Definicja: Obraz Dockerowy to szablon, z którego można utworzyć kontener. Zawiera system plików i parametry do uruchomienia aplikacji.

Przykład: ubuntu:20.04 to obraz bazowy systemu Ubuntu w wersji 20.04.

4. Znacznik Obrazu (Docker Image Tag):

Definicja: Znacznik obrazu to etykieta przypisana do konkretnego obrazu, umożliwiająca identyfikację konkretnej wersji.

Przykład: W ubuntu:20.04, 20.04 to znacznik wersji.

Korzystanie z Rejestrów, Repozytoriów i Znaczników w Praktyce:

1. Pobieranie Obrazu:

Aby pobrać obraz z rejestracji, użyj polecenia docker pull. Na przykład:

docker pull ubuntu:20.04

2. Wysyłanie Obrazu do Rejestru:

Aby udostępnić obraz publicznie lub prywatnie, można go wysłać do odpowiedniego rejestru. Przykład wysyłania do Docker Hub:

docker tag moj_obraz:moj_znacznik nazwa_uzytkownika/repozytorium:moj_znacznik
docker push nazwa_uzytkownika/repozytorium:moj_znacznik

3. Tworzenie Własnego Obrazu ze Znacznikiem:

Aby zbudować i oznaczyć własny obraz z określonym znacznikiem, użyj polecenia docker build. Przykład:

docker build -t moj_obraz:moj_znacznik .

4. Użycie Znaczników w docker-compose.yml:

W pliku docker-compose.yml można używać znaczników do określenia wersji obrazów w kontenerach.


services:
  moj_serwis:
    image: nazwa_uzytkownika/repozytorium:moj_znacznik

5. Używanie Wielu Znaczników:

Można używać wielu znaczników, aby identyfikować różne wersje tego samego obrazu, co ułatwia zarządzanie wersjami i środowiskami.

Umieszczanie własnych obrazów w rejestrach Docker Hub

1. Zaloguj się do Docker Hub:

Jeśli nie masz jeszcze konta na Docker Hub, zarejestruj się na https://hub.docker.com/.

Następnie zaloguj się do konta przy użyciu polecenia:

docker login

Podaj swoją nazwę użytkownika, hasło i adres e-mail.

2. Zbuduj swój obraz:

Przejdź do katalogu, gdzie znajduje się plik Dockerfile Twojego projektu.

Użyj polecenia docker build, aby zbudować swój obraz. Przykład:

docker build -t nazwa_uzytkownika/moj_obraz:moj_znacznik .

Pamiętaj, aby zastąpić nazwa_uzytkownika, moj_obraz i moj_znacznik odpowiednimi nazwami.

3. Oznacz swój obraz:

Użyj polecenia docker tag, aby przypisać odpowiednią nazwę i znacznik Twojemu obrazowi:

docker tag nazwa_uzytkownika/moj_obraz:moj_znacznik nazwa_uzytkownika/moj_obraz:latest

Oznaczanie jako latest jest opcjonalne i używane, gdy chcesz wskazać najnowszą wersję.

4. Wysyłanie obrazu do Docker Hub:

Użyj polecenia docker push, aby przesłać swój obraz do Docker Hub:

docker push nazwa_uzytkownika/moj_obraz:moj_znacznik

To polecenie przesyła obraz do Twojego repozytorium na Docker Hub.

5. Sprawdzenie na Docker Hub:

Zaloguj się na Docker Hub.

Znajdź swoje repozytorium i sprawdź, czy obraz pojawił się w rejestrze.

6. Udostępnianie publicznie lub prywatnie:

Jeśli chcesz, aby Twój obraz był publicznie dostępny, nie musisz podejmować dodatkowych działań.

Jeśli chcesz, aby był prywatny, zaloguj się na stronie Docker Hub, przejdź do ustawień repozytorium i zmień widoczność. Teraz twój obraz jest dostępny na Docker Hub, a inni mogą go pobrać i używać. Pamiętaj o bezpieczeństwie i ogranicz dostęp do prywatnych obrazów przy użyciu odpowiednich ustawień widoczności i kontroli dostępu na Docker Hub. Biorąc pod uwagę powyższe kroki, umieszczanie własnych obrazów w rejestrach Docker Hub staje się prostym zadaniem, a Docker Hub stanowi popularne miejsce do przechowywania i udostępniania obrazów Dockerowych.
