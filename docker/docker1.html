Namespace'y

  • Izolacja - Zapewniają izolację zasobów i środowisk dla każdego kontenera, chroniąc je przed wzajemną ingerencją i złośliwym oprogramowaniem.
  • Rodzaje:
    • Przestrzeń procesów (Pid Namespace)
    • Przestrzeń sieciowa (Network Namespace)
    • Przestrzeń systemu plików (Mount Namespace)

cgroups (Control Groups)

  • Zarządzanie zasobami - Umożliwiają kontrolę i ograniczanie zasobów, takich jak CPU, pamięć RAM, dysk itp., dla każdego kontenera.
  • Efektywność - Zapewniają optymalne wykorzystanie zasobów systemowych i zapobiegają przeciążeniu.

Komponenty Docker

  • Docker Daemon
    • Proces tła zarządzający operacjami na kontenerach.
    • Komunikuje się z Docker API i przyjmuje polecenia od użytkowników.
  • Docker Client
    • Interfejs CLI lub GUI umożliwiający interakcję z Docker Daemon.
    • Wysyła polecenia do Docker Daemon, aby zarządzać kontenerami i obrazami.

Obraz Docker

  • Szablon - Zawiera wszystkie elementy niezbędne do uruchomienia aplikacji, takie jak kod, zależności, konfiguracja i system operacyjny.
  • Lekkość i przenosność - Działa jako szablon, z którego można tworzyć wiele kontenerów, co zapewnia lekkość i przenosność aplikacji.

Proces uruchamiania kontenera

  1. Użytkownik wydaje polecenie Docker CLI, np. `docker run`, określając obraz i opcje.

Zanim zaczniesz

Czym jest kontener?

Kontener to jednostka oprogramowania, która zawiera wszystko, co jest potrzebne do uruchomienia aplikacji, wraz z kodem, środowiskiem wykonawczym, bibliotekami, zależnościami i konfiguracją. Kontenery są izolowane od siebie i od systemu operacyjnego, co pozwala na przenośność aplikacji między różnymi środowiskami.

Dlaczego kontenery podbijają świat?

Kontenery zdobywają popularność ze względu na ich zdolność do izolacji aplikacji, przenośności pomiędzy środowiskami, szybkiego wdrażania, skalowania i efektywnego zarządzania cyklem życia aplikacji, co czyni je atrakcyjnym rozwiązaniem dla organizacji praktykujących DevOps i dążących do ciągłej dostawy oprogramowania.

Jak Docker wykonuje kontenery?

Docker wykonuje kontenery przy użyciu technologii konteneryzacji, której kluczowe elementy to namespace'y i cgroups.

  1. Namespace'y:
    • Pid Namespace: Izoluje przestrzeń procesów, dzięki czemu procesy w jednym kontenerze są odizolowane od procesów w innych kontenerach i na hoscie.
    • Network Namespace: Zapewnia izolację przestrzeni sieciowej, co oznacza, że interfejsy sieciowe, tabele routingu itp. są odseparowane dla każdego kontenera.
    • Mount Namespace: Izoluje przestrzeń systemu plików, umożliwiając, aby kontenery miały własne systemy plików, niezależne od hosta i innych kontenerów.
  2. cgroups (Control Groups): Cgroups umożliwiają kontrolę i ograniczenia zasobów, takich jak CPU, pamięć RAM, dysk itp., które mogą być przypisane do kontenerów. Pozwala to na efektywne zarządzanie zasobami systemowymi.
  3. Docker Daemon: Docker Daemon to tło procesu, który zarządza operacjami na kontenerach. Przyjmuje polecenia od użytkownika, komunikuje się z Docker API i zarządza cyklem życia kontenerów.
  4. Docker Client: Docker Client to interfejs linii komend lub interfejs graficzny, który pozwala użytkownikowi komunikować się z Docker Daemon, wysyłając polecenia i odbierając wyniki.
  5. Docker Image: Docker Image to szablon, na podstawie którego tworzone są kontenery. Zawiera wszystko, co jest potrzebne do uruchomienia aplikacji, w tym kod, zależności, konfigurację itp.

Proces wykonania kontenera w skrócie wygląda tak:

  1. Użytkownik wydaje polecenie Docker Clienta, na przykład docker run.
  2. Docker Client komunikuje się z Docker Daemon, który przetwarza to polecenie.
  3. Docker Daemon sprawdza, czy istnieje odpowiedni Docker Image. Jeśli nie, pobiera go z Docker Hub lub innych źródeł.
  4. Daemon tworzy nowy kontener na podstawie obrazu, konfiguruje namespace'y i cgroups, alokuje zasoby, a następnie uruchamia procesy wewnątrz kontenera.
  5. Procesy w kontenerze są izolowane od innych procesów na systemie, a kontener ma własne środowisko wykonawcze i system plików.

Dzięki tym mechanizmom Docker osiąga izolację i przenośność, co pozwala na łatwe uruchamianie aplikacji w różnych środowiskach.
