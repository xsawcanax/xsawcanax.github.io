Co to jest Docker?

Docker to narzędzie do konteneryzacji aplikacji. Pozwala na łatwe i szybkie uruchamianie aplikacji w izolowanych środowiskach.

  • Zanim zaczniesz
  • Uruchomienie aplikacji w kontenerze, nawiązywanie połączenia z kontenerem
  • Tworzenie własnych obrazów Dockera
  • Pakowanie aplikacji w formie kodu źródłowego do obrazu Dockera
  • Udostępnianie obrazów w Docker Hub oraz innych rejestrach
  • Przechowywanie woluminów Dockera do trwałego przechowywania danych

Podstawowe komendy Dockera

Komenda Opis
docker container run {nazwa_obrazu} Uruchamia nowy kontener z określonego obrazu.
docker ps Wyświetla listę uruchomionych kontenerów.
docker container run -d {nazwa_obrazu} Uruchamia kontener w tle (tryb detached).
docker system df Wyświetla informacje o zużyciu miejsca na dysku przez Docker na systemie.
docker image ls Wyświetla listę dostępnych obrazów na systemie.
docker build -t {nazwa_obrazu} {ścieżka_do_katalogu_z_Dockerfile} Buduje obraz Docker na podstawie pliku Dockerfile.
docker container rm {id_kontenera} Usuwa kontener o podanym ID.
docker image rm {nazwa_obrazu} Usuwa obraz o podanej nazwie.
docker container inspect {id_kontenera} Wyświetla szczegółowe informacje o kontenerze o podanym ID.
docker exec -it {id_kontenera} /bin/bash Loguje się do działającego kontenera.
docker start {id_kontenera} Wznowienie działania zatrzymanego kontenera o określonym identyfikatorze.
docker stop {id_kontenera} Zatrzymuje działanie kontenera o określonym identyfikatorze.

Przydatne zasoby

  • Oficjalna dokumentacja Dockera
  • Docker Hub - repozytorium obrazów Dockera
  • Docker Compose - narzędzie do zarządzania wieloma kontenerami jednocześnie

Podsumowanie

Docker to bardzo przydatne narzędzie dla developerów i administratorów systemów. Dzięki niemu można łatwo i szybko uruchamiać aplikacje w izolowanych środowiskach, co pozwala uniknąć problemów zależności między aplikacjami.
